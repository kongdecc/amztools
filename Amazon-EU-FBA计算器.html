说明 亚马逊欧洲站 FBA 费用政策说明

适用范围：本文档用于说明本工具所采用的 EU 费用计算口径（DE / 欧洲配送网络 FR/IT/ES/PL/CZ / 弃置与退仓） 与费率常量。

说明：亚马逊政策可能更新；如费率或规则发生调整，应以最新官方公告与卖家平台信息为准。

1. 输入字段与计费口径

1.1 尺寸与重量

  • 单位重量（g）：商品包装后的实际称重重量。
  • 三边尺寸（cm）：最长边、次长边、最短边。

1.2 体积重量（g）

计算公式：

  • 体积重量(g) = 长(cm) × 宽(cm) × 高(cm) × 1000 ÷ 5000

1.3 计费重量（g）

  • 计费重量(g) = MAX(单位重量(g), 体积重量(g))

在本工具中： - 尺寸分段、配送费、退仓/弃置费用等均基于上述计费重量与尺寸阈值判断。

2. FBA 尺寸类别判定（EU通用）

本工具会根据 单位重量、体积重量、三边尺寸 判定 FBA 尺寸类别。规则按优先顺序匹配（命中即停止）。

注：以下为“逻辑结构说明”，阈值与类别名称与本工具一致。

2.1 信封类（按单位重量 + 尺寸）

  • 轻型信封 / 标准信封 / 大信封 / 超大信封

典型限制： - 最长边 ≤ 33cm - 次长边 ≤ 23cm - 最短边 ≤ 2.5cm / 4cm / 6cm（不同信封档位不同） - 单位重量上限：100g / 460g / 960g（不同档位不同）

2.2 包裹类（按单位重量 + 体积重量 + 尺寸）

  • 小包裹
  • 标准包裹

典型限制（示例）： - 小包裹：单位重量 ≤ 3900g、体积重量 ≤ 2100g、尺寸 ≤ 35×25×12cm - 标准包裹：单位重量 ≤ 11900g、体积重量 ≤ 7960g、尺寸 ≤ 45×34×26cm

2.3 大件类（按单位重量 + 体积重量 + 尺寸）

  • 小号大件
  • 轻型标准大件
  • 重型标准大件
  • 大型标准大件
  • 超大件
  • 重型大件

大件类通常使用更宽松的尺寸阈值，但配合更高的重量/体积上限。

3. 配送费（FBA配送费 / Unit）

本工具支持两套费用表：

  • DE：亚马逊物流配送费（德国站）
  • 欧洲配送网络：FR / IT / ES / PL / CZ

两套费用表共享同一套“尺寸类别判定”，但 配送费阶梯与常量不同。

3.1 DE 配送费阶梯（摘要）

DE 的配送费按“尺寸类别 + 重量区间”阶梯计算：

  • 信封类：更多使用 单位重量 分段（例如 0–20g、20–40g…）
  • 包裹/大件类：更多使用 计费重量（MAX重量） 分段（例如 ≤150g、150–400g… 或以 760g 为首重并按每 1000g 进位加价）

3.2 欧洲配送网络（FR/IT/ES/PL/CZ）配送费阶梯（摘要）

逻辑与 DE 类似： - 同样按“尺寸类别 + 重量区间”阶梯 - 但各区间对应的费率常量不同（整体水平明显高于 DE 的示例）

4. 汇率换算

本工具支持将 EUR 金额换算为 RMB：

  • RMB = EUR × 汇率（EUR/RMB）

注意：汇率由用户输入，本工具不自动联网获取实时汇率。

5. 月度仓储费（Storage）

5.1 月度仓储费率（EUR）

费率由两个因素决定：

  • 是否旺季（是/否）
  • 是否大件（尺寸类别名称包含“大件”）

本工具中的费率常量如下：

  • 旺季：
    • 大件：34.49
    • 非大件：52.20
  • 非旺季：
    • 大件：21.78
    • 非大件：27.54

注：此处为本工具内置的费率常量口径，不代表官方最新值。

5.2 月实际仓储费（EUR）

  • 月实际仓储费(EUR) = 月度仓储费率 × 存放天数 × 在库数量 × 体积(m³) ÷ 当月总天数
  • 其中：体积(m³) = 长×宽×高 ÷ 1,000,000（cm³ → m³）

6. 仓储利用率附加费（Storage Utilization Surcharge，仅DE模块展示）

按“利用率周数”分段加收。本工具使用如下分段常量：

  • ≤22 周：0
  • 22–28 周：14.85
  • 28–36 周：27.05
  • 36–44 周：37.61
  • 44–52 周：46.55

  • 52 周：76.87

附加费计算结构为：

  • 附加费 = 基础体积占用（按天数/数量/体积/当月天数折算）× 分段费率常量

7. 超龄库存附加费（Aged Inventory Surcharge，仅DE模块展示）

按“仓储天数段”分段加收，分段常量如下：

  • 0–240：0
  • 241–270：47.99
  • 271–300：125.16
  • 301–330：129.92
  • 331–365：135.98
  • 366–455：MAX(227.63, 超龄数量×0.2)
  • 455+：MAX(261.97, 超龄数量×0.25)

附加费计算结构为：

  • 超龄附加费 =（存放天数 × 超龄数量 × 体积 ÷ 当月天数）× 分段常量

8. 弃置 / 退仓费（Removal / Disposal / Return to Seller）

本工具支持三种退货类型：

  • 退回给卖家
  • 清仓
  • 弃置

并分为两套费率：

  • 标准尺寸费率
  • 大件尺寸费率

8.1 计费尺寸判定

默认口径可按尺寸类别自动判断： - 尺寸类别包含“大件” → 使用大件尺寸费率 - 否则 → 使用标准尺寸费率

同时支持手动选择“标准尺寸费率/大件尺寸费率”。

8.2 标准尺寸：单个费用（EUR）

标准尺寸按计费重量（g）分段（≤200 / ≤500 / ≤1000 / >1000，并可能按每1000g进位加价）。

8.3 大件尺寸：单个费用（EUR）

大件尺寸按计费重量（g）分段（≤500 / ≤1000 / ≤2000 / ≤5000 / >5000，并可能按每1000g进位加价）。

9. 利润试算（口径说明）

用于估算单品在不同站点口径下的利润指标（示例：DE / 欧洲配送网络）。

输入： - 售价（EUR） - 成本（RMB） - 汇率（EUR/RMB） - 佣金率（%）

计算： - 成本（EUR）= 成本（RMB）÷ 汇率 - 佣金（EUR）= 售价 × 佣金率 - 毛利（EUR）= 售价 − 成本(EUR) − FBA配送费(EUR) − 佣金(EUR) - 毛利率 = 毛利 ÷ 售价

10. 免责声明与使用提示

  • 本文档为费用计算说明与口径解释，便于理解计算结果；不构成任何形式的官方承诺或法律/财务建议。
  • 实际收费可能随站点政策、活动、时间、产品属性与后台配置而变化；请以亚马逊卖家平台/官方公告为准。
  • 输入的汇率由用户自行提供；若使用实时汇率或特定结算汇率，请以你的业务口径为准。

11. 常见计算案例

以下案例用于演示计算口径与字段含义，数值为示例输入；实际结果会随费率、汇率、旺季等变化。

案例 A：标准包裹（DE）配送费 + 仓储费

  • 输入
    • 单位重量：1.95 g（示例值，实际业务中通常为 1950 g；以实际称重口径为准）
    • 尺寸：39 × 22 × 15 cm
    • 汇率：8.0884
    • 旺季：否（或自动判断）
    • 仓储：存放 10 天、当月 30 天、在库 100
  • 过程（摘要）
    • 体积重量 = 39×22×15×1000/5000 = 2574 g
    • 计费重量 = MAX(单位重量, 体积重量) = 2574 g
    • 尺寸类别：标准包裹
    • 配送费：按“尺寸类别 + 重量区间”查询
    • 仓储费：按体积(m³)、天数、数量折算

案例 B：欧洲配送网络 vs DE 的差异

  • 同一组尺寸/重量输入，在不同站点口径下：
    • 尺寸类别通常一致（判定逻辑相同）
    • 配送费可能不同（费率常量不同）
    • 因此利润试算中，净利润/净利率也会出现差异

案例 C：弃置 / 退回给卖家（标准尺寸）

  • 输入
    • 退货类型：退回给卖家
    • 计费重量：按 MAX(单位重量, 体积重量)
    • 数量：100
  • 结果
    • 单个费用按“标准尺寸费率”的重量分段计算
    • 总费用 = 单个费用 × 数量

12. FAQ（常见问题）

Q1：为什么同样输入，DE 与欧洲配送网络的结果不同？

  • 站点口径的配送费费率常量不同；尺寸类别判定逻辑一致，但查表结果不同。

Q2：旺季如何判断？

  • 本工具可按当前日期自动判断旺季（规则可配置）。如平台定义的旺季窗口与工具默认规则不一致，应以平台口径为准。

Q3：重量输入用 kg/lb/oz 可以吗？

  • 可以。本工具会将输入重量统一换算为 g 后参与计算。

Q4：体积重量与计费重量有什么区别？

  • 体积重量：由尺寸推导出的“等效重量”。
  • 计费重量：单位重量与体积重量取较大者，用于阶梯计费。

Q5：利润试算里的“成本”包含哪些？

  • 支持拆分为采购成本与头程/运费（以 RMB 输入），并按输入汇率换算为 EUR 参与计算。
